4) The value of the dereference operation *messagePtr would be, 'H'. From what I've read the OS finds a contiguous block of 12 bytes in memory to store it, however, the pointer messagePtr is only 8 bytes long itself - it cant hold the whole string. Instead, it only stores the Base Address (the memory location of the very first byte) which happens to be 8bytes long, itself. Because the pointer only knows where the string starts it grabs the single character sitting there. I also found that arrays and pointers are virtually the same thing under the hood, so to speak, in C. The compiler treats the square brackets as just a readable shortcut for dereferencing. In other words, these two statements are identical to a CPU: messagePtr[0] and *messagePtr.
