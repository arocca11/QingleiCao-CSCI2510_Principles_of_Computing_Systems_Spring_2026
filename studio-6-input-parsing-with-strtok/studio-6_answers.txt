2)
The difference between the initial call to strtok() and subsequent calls is that the first call get the string and the subsequent calls get NULL to pick up where it left off. So the first call serches from the beginning of buffer, finds the first word, marks the end of the word with \0 and remembers that spot in memory and hands back the first word. The subsequent calls just get NULL which tell strtok() to look at the \0 bookmark and pick up exactly where we left off. it will then find the second word, move /0 and hand it back.
